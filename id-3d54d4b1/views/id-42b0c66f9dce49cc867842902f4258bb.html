SV-Reference Data [Motivation] ()
NARRATIVE: The SV-Reference Data [Motivation] view extends Semantic view and provides a comprehensive framework for leveraging various elements and their relations to enhance the motivation and understanding of the system. The elements in the JSON are grouped into three main categories: Business Agnostic Reference Data Solutions, Business Agnostic Business Information Base, and Customs Reference Data. The Business Agnostic Reference Data Solutions group contains a catalogue of data objects that are relevant for the EU labour market and education and training. This includes the ESCO classification, which identifies and categorises skills, competences, and occupations, and the Regulated Professions DB, which contains information on regulated professions, statistics on migrating professionals, contact points and competent authorities. This group also includes the Business Agnostic EU Vocabularies Reference Data Solutions Catalogue, which contains data objects like NUTS, Reference Data Asset, Authority Tables, and the Digital Europe Thesaurus. The Business Agnostic Business Information Base is a diagram model-group that contains the Data Quality business object, which provides information on data quality. The Customs Reference Data group contains a catalogue of data objects that are relevant for customs operations. This includes the Reference Data Customs Offices List, Harmonized System (HS) Codes, ISO Country Codes, Currency Code List, Unified Code for Units of Measure (UCUM), Legal and Regulatory References, and Customs Valuation Methods. This group also includes the Customs Information Base, which contains business objects like the Economic Operators Registration and Identification number (EORI), Imports information, Exports information, Customs partners Information, Combined nomenclature, Economic value Information, Classification of goods Information, Origin of goods Information, and Economic Operators Information. The Reference Data is a data object that enables the definition of a set of values or codes that are used to categorize, classify, or describe data elements in a consistent and standardized way. The Diagram Model-connection ABB is an element that describes the relationships between the different elements in the system. However, the JSON does not provide any specific details about this element.
