SV-Customs Datasets [Motivation] ()
NARRATIVE: The SV-Customs Datasets [Motivation] view extends Semantic view and provides a comprehensive framework for leveraging various elements related to customs data. This view is composed of four main elements: "Customs Dataset Catalogue", "Customs Information Base", "Dataset", and "Customs Dataset Solutions". The "Customs Dataset Catalogue" is a Grouping ABB that consists of multiple Data Object ABBs. These include "Customs statistical metadata on goods imported", "Applications and Requests metadata", "Laboratory reports index", "Tariffs metadata", "Risk metadata", "Origins metadata", "Cargo metadata", "Declarations metadata", "Intra-EU purchases and sales metadata", "Goods taxonomy metadata", "Notifications metadata", "Procedures metadata", "Customs statistical metadata on goods exported", "Goods metadata", and "Laboratories testing metadata". Each of these data objects represents a specific type of metadata related to customs operations. The "Customs Information Base" is a Diagram Model-group ABB that contains various Business Object ABBs. These include "Economic Operators Registration and Identification number (EORI)", "Free circulation requirement information", "Laboratories information", "Imports information", "Combined nomenclature", "Special Procedures Information", "Customs partners Information", "Exports information", "Binding Tariffs Information", "Workforce information", "Economic value Information", "Internal logistic Information", "Origin of goods Information", "Guarantee Information", "Classification of goods Information", "Threads and fraudulent actions Information", "Economic Operators Information", "Temporary storage Information", and "Duties calculation Information". Each of these business objects represents a specific type of information related to customs operations. The "Dataset" is a Data Object ABB that represents a collection of related data organized and presented in a structured format. This element provides a general representation of datasets in the context of customs operations. The "Customs Dataset Solutions" is a Grouping ABB that includes two Data Object SBBs: "EORI metadata" and "Economic operators metadata". These data objects represent specific types of metadata related to customs dataset solutions. The relationships between these elements are not explicitly defined in the JSON. However, it can be inferred that the "Customs Dataset Catalogue" and "Customs Dataset Solutions" are related to the "Dataset" as they provide specific types of metadata for datasets. Similarly, the "Customs Information Base" can be related to the "Dataset" as it provides specific types of information that can be included in datasets. The motivation behind these elements and their relationships is to provide a comprehensive and structured representation of customs data, which can be leveraged for various customs operations and solutions.
